MITM-DomainFronting is a local proxy technique that lets you access certain Google services directly from your device — no remote VPN server or worker required. It works by intercepting your browser's TLS traffic locally, then forwarding it to Google's servers using a spoofed SNI, bypassing censorship filters that block domains by name. The project is built on Xray-core and is available for Windows, Linux, macOS, and Android (no root needed).

## What you can access

MITM-DomainFronting currently enables access to Google's main services that share Google's core infrastructure. This includes services such as **Google Meet**, **Google Drive**, and similar Google products routed through Google's primary servers.

Some Google services are not accessible through this method:

- **YouTube videos** — YouTube's video content uses its own CDN infrastructure (`googlevideo.com`) with separate routing. While the domain is matched in routing rules, video playback may not work fully.
- **Gemini** — Google has applied IP-level blocks on Iranian IP addresses for Gemini, so domain fronting cannot bypass this restriction.

<Note>
  The list of accessible services may expand as the project is updated. Check the project repository for the latest information.
</Note>

## Platform support

The method runs entirely on your local device and is supported on all major platforms:

| Platform | Client app | Notes |
|----------|-----------|-------|
| **Windows** | v2rayN | Full system proxy or per-browser |
| **Linux** | v2rayN or compatible | System proxy |
| **macOS** | v2rayN or compatible | System proxy |
| **Android** | v2rayNG | Browser-only on non-rooted devices; Hev TUN must be enabled |

<Info>
  On non-rooted Android, only browsers can use this method. Standalone apps typically cannot route through it.
</Info>

## Requirements

Before you can use MITM-DomainFronting, you need the following:

- **v2rayN** (Windows/Linux/macOS) — download from the [v2rayN releases page](https://github.com/2dust/v2rayN/releases), or **v2rayNG** (Android) — download from the [v2rayNG releases page](https://github.com/2dust/v2rayNG/releases)
- **Xray-core v26.2.6 or later** — the minimum version is enforced by the `MITM-DomainFronting.json` config itself via the `"min": "26.2.6"` version field
- **A personal self-signed certificate** — two files: `mycert.crt` and `mycert.key`, generated using the included `certificate_generator.bat` script (Windows) or an equivalent tool

<Warning>
  Never share your `mycert.key` private key file with anyone, and never use a certificate (`mycert.crt`) generated by someone else. Doing so would allow that person to intercept your traffic. Generate your own certificate and keep both files private.
</Warning>

## Native integration in Xray-core

MITM-DomainFronting started as a standalone project and was subsequently integrated directly into Xray-core as a native feature. This means you can use it with a standard v2ray/Xray config file without any additional tooling. The integration was tracked in [XTLS/Xray-core issue #4348](https://github.com/XTLS/Xray-core/issues/4348).

<Tip>
  Because the method is built into Xray-core, initial setup involves a few steps (certificate generation, certificate trust, and config import), but day-to-day use is a simple on/off toggle in your client app.
</Tip>
